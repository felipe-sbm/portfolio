## Contexto

Projeto voltado para acelerar revisão técnica e reduzir gargalos em times de desenvolvimento.

## O que foi construído

- Pipeline para análise de diffs e arquivos alterados
- Geração de feedback técnico com foco em bugs e riscos
- Regras para priorizar achados por severidade
- Saída padronizada para facilitar leitura por desenvolvedores

## Stack

- Python
- Integrações com APIs de LLM
- Estrutura orientada a workflows de CI/CD

## Aprendizados

O maior ganho foi modelar prompts e critérios de revisão de forma consistente, evitando respostas genéricas e aumentando a utilidade prática do agente.
