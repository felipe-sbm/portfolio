## Contexto

A **Insomnia Machine** nasceu como um experimento de IoT para unir eletrônica, software embarcado e experiência de usuário em um protótipo funcional de cafeteria automática.

## O que foi construído

- Controle de preparo por estados (idle, heating, brewing)
- Leitura de sensores para nível de água e temperatura
- Acionamento de bomba e resistência com regras de segurança
- Base de telemetria para avaliar consistência do preparo

## Stack

- C++
- ESP32
- Sensores analógicos e digitais
- Comunicação serial para debug

## Aprendizados

Esse projeto consolidou conceitos importantes de sistemas embarcados: tolerância a falhas, debouncing, controle de tempo sem bloqueio e clareza de arquitetura para firmware.
